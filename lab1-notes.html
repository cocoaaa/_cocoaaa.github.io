by Hae Jin Song
in 6s08 on 2016-01-27

  1. Flash memory is a type of ROM

    • Teensy has 256k
    • Compile your program and upload it to the microcontroller -> the program is saved in the flash memory -> CPU brings in ("reads") and executes the program
    • Ref: HowStuffWorks

  2. Serial port

    • a serial communication interface through which information transfers in/out one bit at a time

    • compare: parallel communication

  3. SPI (Serial Peripheral Interface)

    • an interface bus commonly used to send data between microcontrollers and small peripherals such as shift registersss, sensors, and SD cards.

  4. I2C-Bus > allows easy communication between components which reside on the same circuit board.

    For example, communication between teensy and the LED screen, or between teensy and the IMU (Inertial Measurement Unit).

  5. Baud rate

    • How many bits per second are communicated
    • Teensy uses USB to communicate, so the baud rate on the serial ports don't matter.

  6. #define varName value

    • The compiler replaces every occurance of the variable with the assigned value. So, we can set up gloabal variables without taking up memory.

  7. Adafruit_SSD1306 display(OLED_RESET)

    • Adafruit_SSD1306 is a class
    • display is the name of the instance
    • OLED_RESET is the initialization argument to create display instance
