Javascript scope hoisting

by Hae Jin Song
in js on 2016-01-27

Source

variable scope

Key: not block scoping: {} does not define a scope, but function scoping

  1. Global variable

    • no var used to declare a variable (both in and outside of a function)

      myname = "Hayley"; //global
function sayHi(){
    console.log("Hi, "+myname); // "Hi, Hayley" rather than "Hi, undefined"
}

    • definted outside of a function with var

  2. Local variable defined with var takes precedence over global variable of the same name when a function is called.

  3. Arguments into a function are also local variable

    var myname ='HAYLEY';
function sayhi(myname){
    console.log(myname);
}
sayhi("sam"); //"sam"
console.log(myname); //"HAYLEY"

So think of var myFunction(arg1){} as var myFunction(){ var arg1=arg1;}

Hoisting

Variable declaration (not initialization or assignment) and function declaration are hoisted to the top of the script or function if defined within a function.
